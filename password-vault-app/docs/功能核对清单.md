# 密拾 APP 功能核对清单

> 对照《密拾APP 项目开发任务书》与当前仓库实现（Capacitor WebView + Android 原生悬浮层）逐项核对。  
> 图例：**✅** 已实现且可用；**⚠️** 部分实现、与任务书有差距或依赖权限/用户操作。

最后更新：2026-06-15

---

## 一、任务书五项硬性核心功能

| # | 任务书要求 | 状态 | 说明 |
|---|------------|------|------|
| 1 | 手动录入/存储账号、密码、网址；本地增删改查 | ✅ | `www/index.html` 提供 PIN 解锁后的账号库；`mishi-ext.js` 扩展网址字段、分类与保存逻辑。 |
| 2 | 后台常驻悬浮球；通栏纸感交互 | ✅ | `OverlayService` 前台服务 + `SYSTEM_ALERT_WINDOW`；设置页可开关。需用户授予「显示在其他应用上层」。 |
| 3 | 截屏自动归档至密拾私密库 | ⚠️ | 监听 `MediaStore` 新截图 → 复制到应用私有目录 `files/vault_shots/`，可选删除系统相册原图（`MediaDeleteHelper`）。**待入库前**文件为明文 JPEG；解锁 APP 后 `syncPendingScreenshots` 才写入 AES 加密 vault。需相册读取权限。 |
| 4 | 悬浮球数据操控（8 项菜单 + 跳转主页） | ⚠️ | 8 个菜单项均已实现（见下节）。**快速搜密码**依赖原生侧搜索索引，但 Web 同步索引不含密码字段，悬浮窗搜索常无法复制密码（见 §二-5）。任务书「用户自定义删除菜单项」**未实现**。 |
| 5 | 本地加密存储；无云端/无账号/无网络同步 | ⚠️ | 主库：`localStorage` + AES-256-GCM（PIN 经 PBKDF2 派生密钥）。Manifest **无** `INTERNET` 权限。⚠️ 设置中仍有**明文 JSON** 导入/导出（`export-btn`/`import-btn`），与任务书「仅加密备份」不完全一致；另有 `.mishi` 加密备份（`mishi-ext.js`）。 |

## 二、悬浮球菜单（8 项）

| 菜单项 | 状态 | 说明 |
|--------|------|------|
| 1 新增账号密钥 | ✅ | 悬浮窗表单 → `VaultCache` 待同步队列 → 解锁 APP 后合并进加密库。 |
| 2 新增网址书签 | ✅ | 同上，类型 `url`。 |
| 3 截屏监听开关 | ✅ | 切换 `VaultCache` 标志；关闭后不再自动拾取。 |
| 4 一键查看最近截图 | ✅ | 展示最近 3 张缩略图（待处理 + `vault_shots` 目录）。 |
| 5 快速搜密码 | ⚠️ | UI 与复制流程已实现；索引由 `syncSearchIndex` 同步且**不含 password**，Overlay 侧 `password` 字段通常为空，功能不完整。 |
| 6 临时私密剪贴板 | ✅ | 文本存 SharedPreferences，默认 5 分钟过期；可覆盖/复制。 |
| 7 悬浮球隐藏 | ✅ | 隐藏 30 秒后自动恢复（非任务书「定时恢复显示」的可配置版，但满足临时隐藏）。 |
| 8 打开 APP 主页 | ✅ | 启动 `MainActivity`。 |

**悬浮球形态（任务书 3.1-3）**

| 要求 | 状态 | 说明 |
|------|------|------|
| 默认可拖拽圆球 | ✅ | 位置持久化 `ScreenshotStore.saveBubblePosition`。 |
| 自定义大小/透明度/颜色 | ⚠️ | `VaultNativePlugin.setBubbleStyle` / `VaultCache` 已支持，**设置页无 UI**，需代码或后续补界面。 |
| 纸感菜单卡片 | ✅ | `OverlayService.showBubbleMenu` 圆角渐变样式。 |
| 用户自定义删除菜单选项 | ⚠️ | **未实现**（菜单固定 8 项）。 |

## 五、数据存储规则（任务书第五章）

| 规则 | 状态 | 说明 |
|------|------|------|
| 无网络库/无上传接口 | ✅ | 无 `INTERNET` 权限；无远程 API 代码。 |
| 敏感数据加密存私有目录 | ⚠️ | 主数据在 WebView `localStorage`（Capacitor 应用私有存储）；截图 pending 阶段为应用内明文文件。 |
| 卸载后数据清除提示 | ⚠️ | 需在「关于」/文案中用户教育；Android 卸载即删应用数据，**无单独卸载向导页**。 |
| 截屏不入系统公开相册（收纳后） | ⚠️ | 尝试删除系统截图；失败时仍可能残留，取决于厂商权限。 |
| 禁止外部直接读密拾文件 | ✅ | 应用 sandbox；FileProvider 仅用于分享导出场景。 |

## 九、建议优先补齐项（开发 backlog）

1. 修复悬浮窗「快速搜密码」：同步索引策略或 Overlay 内解密查询（避免明文长期落盘）。  
2. 移除或加警告的明文 JSON 导出，统一为 `.mishi`。  
3. 设置页暴露悬浮球大小/透明度/颜色。  
4. 待入库截图加密或缩短 pending 明文窗口。  
5. 评估 `android:allowBackup` 与任务书「卸载即毁」表述的一致性。

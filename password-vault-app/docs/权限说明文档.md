# 密拾 APP 权限说明文档

本文说明 **密拾**（包名 `com.personal.passwordvault`）在 Android 端声明与使用的权限、用途、拒绝后果，以及**无联网权限**策略。

配置来源：`android/app/src/main/AndroidManifest.xml` 与 Capacitor 插件 `@CapacitorPlugin` 声明。

---

## 1. 权限总览

| 权限 | 类型 | 是否必需 |
|------|------|----------|
| `SYSTEM_ALERT_WINDOW` | 特殊（设置页授予） | 悬浮球/悬浮菜单/悬浮对话框：**可选** |
| `FOREGROUND_SERVICE` | 普通 | 开启悬浮球后台能力时使用 |
| `FOREGROUND_SERVICE_SPECIAL_USE` | 普通（Android 14+ 子类型） | 同上，子类型 `screenshot_capture_overlay` |
| `POST_NOTIFICATIONS` | 运行时（Android 13+） | 前台服务通知：**建议授予** |
| `READ_MEDIA_IMAGES` | 运行时（Android 13+） | 自动识别系统截图：**可选** |
| `READ_EXTERNAL_STORAGE`（maxSdk 32） | 运行时（Android 12 及以下） | 同上 |

**未声明且刻意不使用的权限（任务书要求）**

- `INTERNET` — **未声明**，应用不发起网络访问。  
- 定位、通讯录、短信、电话、麦克风、相机、蓝牙等 — **均未声明**。

## 2. 逐项说明

### 2.1 显示在其他应用上层（`SYSTEM_ALERT_WINDOW`）

- **用途**：绘制悬浮球、展开纸感菜单、在其它 App 上方显示录入/搜索/剪贴板对话框（`TYPE_APPLICATION_OVERLAY`）。  
- **如何授予**：系统「特殊应用访问」→「显示在其他应用的上层」→ 允许密拾；APP 内设置项可跳转 `Settings.ACTION_MANAGE_OVERLAY_PERMISSION`。  
- **拒绝时**：无法显示悬浮球与悬浮窗功能；**主 APP 内密码库、PIN、加密备份仍可使用**。  
- **关闭方式**：系统设置中关闭，或在密拾设置中关闭「后台悬浮球」开关（停止 `OverlayService`）。

### 2.2 前台服务（`FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_SPECIAL_USE`）

- **用途**：在用户开启悬浮球后保持 `OverlayService` 运行，避免被系统轻易杀死；通知栏显示低优先级常驻通知「密拾后台…」。  
- **拒绝/限制时**：Android 可能限制后台运行，导致截屏监听或悬浮球**间歇失效**；用户可从最近任务划掉应用，服务会尝试在仍开启开关时恢复（`onTaskRemoved`）。  
- **与耗电**：仅在使用者主动开启悬浮球时运行；关闭开关即 `stopSelf()`。

### 2.3 通知（`POST_NOTIFICATIONS`，Android 13+）

- **用途**：展示上述前台服务通知（法律/系统要求可见通知）。  
- **拒绝时**：部分系统仍可能允许 FGS 但隐藏通知，或限制服务；可能导致悬浮能力不稳定。不影响 APP 内解锁使用。

### 2.4 读取图片 / 存储（`READ_MEDIA_IMAGES` 或 `READ_EXTERNAL_STORAGE`）

- **用途**：通过 `MediaStore` 监听新产生的**截图**文件，复制到应用私有目录并可选删除相册副本；**不**扫描用户全部相册做上传或分析。  
- **如何请求**：设置中开启悬浮球时会调用 `VaultNativePlugin.requestMediaPermission()`。  
- **拒绝时**：无法自动检测系统新截图；Toast 提示「请授予相册读取权限…」。用户仍可在 APP 内**手动**管理已入库截图、使用密码库与悬浮窗手动录入（不依赖相册权限的部分）。  
- **关闭方式**：系统应用权限设置中撤销「照片和视频」/「存储」。

## 3. 非 Manifest 权限能力说明

以下能力通过系统 API 完成，**不在 Manifest 中单独声明**，但仍涉及用户数据：

| 能力 | 说明 |
|------|------|
| 剪贴板读写 | 悬浮窗「复制密码」、临时剪贴板使用系统 `ClipboardManager`；数据仅在本地进程。 |
| 打开网址 | `Intent.ACTION_VIEW` 调起浏览器；不经过密拾服务器。 |
| 导出截图到相册 | `MediaStore` 写入 `Pictures/密拾`；用户主动触发。 |
| WebView 本地存储 | 加密 vault 存于应用私有 WebView 存储，其它应用不可读。 |

## 4. 无联网（No Internet）策略

1. **Manifest 未声明 `INTERNET`**，默认无法建立一般 outbound 网络连接（Capacitor WebView 加载本地 `www` 资源）。  
2. **无**用户注册、登录、云备份、广告 SDK、统计上报等代码路径。  
3. 备份均为**用户手动**导出到本地文件（`.mishi` 或 JSON），由文件管理器/聊天工具分享时离开设备，**非 APP 自动上传**。  
4. 开发打包时 `build-apk.bat` 会下载 Gradle/Maven 依赖，属于**开发机联网**，与安装到手机后的运行态无关。

---

## 5. 权限最小化与任务书对齐

- 任务书要求：悬浮窗、截屏读取、后台运行、本地加密存储；**禁止**多余隐私权限。  
- 当前实现与之一致；可选权限均可手动关闭且不影响核心本地 vault（除对应特性本身）。  
- 建议在首次引导与设置页保留简短说明，便于应用商店审核与用户信任。

## 6. 用户常见问题

**Q：为什么密拾要「显示在上层」？**  
A：为了在刷剧/游戏时通过悬浮球快速记密码或收截图，不强制要求。

**Q：会不会上传我的相册？**  
A：不会。读取权限仅用于识别**新截图**路径并复制到应用私有目录，无网络权限上传。

**Q：关闭所有权限还能用吗？**  
A：可以当作离线 PIN 密码本使用；自动收截图与悬浮球不可用。
